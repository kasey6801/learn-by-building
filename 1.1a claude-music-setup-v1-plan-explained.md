# Claude + Apple Music Setup — V1 Plan Explained

> A macOS installer. Double-click one file and Claude Desktop gains the ability to read your Apple Music library, search for tracks, and build historically-informed playlists — all in plain English.

---

## What this is, in plain terms

This project is a single installer file — `Claude Music Setup.command` — that you run once on your Mac. It takes about two minutes and requires no technical knowledge. When it finishes, you can open Claude Desktop and ask it things like:

*"Build a playlist of the 20 most sampled songs from the 1970s that were sampled in 80s and 90s hip hop. Call it 70s Hip Hop Samples."*

Claude will search its knowledge of music history, find the matching tracks in your Apple Music library, and create the playlist automatically. If a track you asked for is not in your library, Claude will tell you which ones are missing and give you Apple Music links so you can add them.

The installer connects Claude Desktop to your Apple Music library using a small background helper called a music connector. Once installed, the connector stays connected silently — you do not need to do anything to use it. If you ever want to disable it, one toggle inside Claude Desktop is all it takes.

Nothing is sent to any third-party service. Claude's language understanding runs through Anthropic's cloud as it always does. The music connector only talks to the Music app on your own Mac.

**Scope of v1:** macOS on Apple Silicon (M1 or later) only. Requires Claude Desktop. Works with songs already in your personal Apple Music library — not the full streaming catalogue.

## Glossary — terms used in this guide

These are words and phrases that appear throughout this document. Reading this section first will make the rest significantly easier to follow.

**AppleScript**
A scripting language built into macOS that allows programs and scripts to control other apps on your Mac. When Claude asks "add this song to a playlist," the music connector translates that into an AppleScript command that the Music app understands and executes — just as if you had clicked and dragged in the app yourself. AppleScript is why the connector requires the macOS Automation permission.

**Apple Silicon**
The generation of chips Apple began making in-house in 2020, starting with the M1. All Macs sold since late 2020 use Apple Silicon. The chip architecture is called `arm64`. Intel-based Macs (pre-2020) use a different architecture (`x86_64`) and are not supported by this installer. You can check which chip your Mac has under Apple menu → About This Mac.

**Architecture (chip)**
The fundamental design of a processor that determines what software it can run. Apple Silicon (`arm64`) and Intel (`x86_64`) are the two architectures used in Macs. Software compiled for one architecture does not run on the other without a translation layer. This installer checks your chip type at startup and stops if it is not Apple Silicon.

**Automation permission**
A macOS privacy setting (found in System Settings → Privacy & Security → Automation) that controls which applications are allowed to control other applications using AppleScript. You must personally grant Claude the Automation permission to control the Music app — this cannot be done by a script, by design. The permission is per-app and per-controlled-app: "Claude is allowed to control Music."

**Bash / shell script**
Bash is the language used to write the installer (`Claude Music Setup.command`). A shell script is a plain text file containing commands that the Terminal executes in sequence — opening apps, downloading files, editing configuration, and so on. The `.command` extension tells macOS to run the file in Terminal when you double-click it. The `#!/bin/bash` line at the top of the file tells the operating system which language the script is written in.

**Config file / JSON config**
A plain text file that stores settings for an application. Claude Desktop reads its settings from a file called `claude_desktop_config.json` stored in a hidden folder on your Mac. The installer adds a single entry to this file — the music server definition — without changing anything else. JSON is the format used: structured text that both humans and programs can read, using curly braces and key-value pairs.

**Connector / MCP connector**
A small background server that gives Claude Desktop the ability to interact with apps or services on your Mac. Without connectors, Claude can only have text conversations. With the music connector installed, Claude gains a set of tools it can call: search your library, create a playlist, add songs to a playlist, and so on. Connectors appear in Claude Desktop under the `+` icon in the chat bar. The technical name for this system is Model Context Protocol (MCP).

**Disk image (.dmg)**
A macOS packaging format that works like a virtual read-only folder. Double-clicking a `.dmg` file mounts it — a new window appears showing the files inside (in this case, the installer and the README). Nothing is installed at this point. The disk image is just a container used to deliver the installer cleanly. Once you have run the installer, the `.dmg` can be ejected and deleted.

**exit / set -euo pipefail**
A safety instruction at the top of the installer script. `set -e` means "stop immediately if any command fails." `set -u` means "treat any undefined variable as an error." `set -o pipefail` means "if a command in a pipe fails, treat the whole pipe as failed." Together these three settings ensure the installer never silently continues past an error — if something goes wrong, it stops and shows a clear error message instead of proceeding in a broken state.

**JSON (JavaScript Object Notation)**
A plain text format for storing structured data. The Claude Desktop configuration file is written in JSON. An example entry looks like: `{ "music": { "command": "npx", "args": ["@pedrocid/music-mcp@latest"] } }`. JSON uses curly braces, colons, and commas in a specific pattern. The installer uses Python to read and write JSON safely, so that the format stays valid regardless of what else might be in the file.

**MCP (Model Context Protocol)**
The open standard that defines how Claude Desktop communicates with connectors. It is a protocol — a set of rules for how messages are formatted and exchanged — so that any connector written to the standard can work with any host application (like Claude Desktop) that supports it. When Claude needs to search your music library, it sends a tool call through the MCP protocol to the music connector, which handles the actual interaction with the Music app.

**MIT license**
A permissive open-source software license. Software released under the MIT license is free to use, inspect, copy, modify, and redistribute. The music connector (`music-mcp`) and this installer are both MIT-licensed, meaning you can read exactly what they do, share them, or modify them. The source code for the music connector is publicly available at github.com/pedrocid/music-mcp.

**Node.js**
A runtime environment that allows JavaScript code to run outside a browser, directly on your computer. The music connector is written in JavaScript and requires Node.js to run. The installer checks whether Node.js is already on your Mac and installs it automatically if not. Node.js version 18 or later is required. The installer fetches version 22 LTS (Long-Term Support), which is the current stable version.

**npx**
A command that comes with Node.js. It downloads and runs a package from the npm registry (npm is the world's largest public library of JavaScript packages) without permanently installing it. The installer configures Claude Desktop to start the music connector using `npx @pedrocid/music-mcp@latest` — meaning every time Claude starts the connector, it fetches the latest published version automatically. No separate update step is ever needed.

**PATH**
A list of folders the operating system searches when you type a command in Terminal. When Node.js is installed, its location (`/usr/local/bin`) needs to be in PATH so that the shell can find the `node` and `npx` commands. The installer temporarily adds `/usr/local/bin` to PATH immediately after installing Node.js, so the rest of the script can use the newly installed binaries without requiring the user to open a new Terminal window.

**pgrep**
A macOS and Linux command that searches the list of running processes by name. The installer uses `pgrep -x "Claude"` to check whether Claude Desktop is currently open. If it is, the installer warns the user that Claude will need to be restarted after setup completes — because config changes only take effect at startup.

**Python 3**
A programming language that ships pre-installed with macOS. The installer uses Python to safely read, modify, and write back the Claude Desktop configuration file. Using Python rather than text manipulation commands (like `sed` or `awk`) ensures the JSON stays structurally valid, handles edge cases like malformed existing files, and creates a timestamped backup if the existing file cannot be parsed.

**`sudo` / Mac password**
`sudo` (short for "superuser do") runs a command with administrator privileges. Installing Node.js requires writing files to `/usr/local`, which is a protected system folder. The installer uses `sudo installer` to run the Node.js package installer with the necessary privileges, which is why macOS prompts for your Mac password at that step. The installer only uses `sudo` for this one command.

**Terminal**
The built-in macOS application that gives you a text interface for running commands directly on your Mac. When you double-click `Claude Music Setup.command`, macOS opens Terminal automatically and runs the script inside it. You see the installer's progress printed as lines of text. You do not need to know how to use Terminal to run this installer — the double-click is the only step required.

**`uname`**
A shell command that returns information about the operating system and hardware. `uname` returns "Darwin" on all macOS versions (Darwin is the BSD-based foundation macOS is built on). `uname -m` returns the CPU architecture: "arm64" for Apple Silicon, "x86_64" for Intel. The installer uses both to confirm the user is on a supported Mac before proceeding.

## Technologies — what each one does and why it is here

### Bash shell script
The installer itself is a bash script — a text file of commands executed in sequence by the Terminal. Bash was chosen because it is the native scripting environment of macOS: it is pre-installed, has no dependencies, can call any system tool directly, and produces readable terminal output. No separate runtime or installation is required to run a bash script. It handles the five-step installation flow, prints colour-coded progress messages, and exits cleanly on any error.

Learn more: [Bash scripting guide — GNU](https://www.gnu.org/software/bash/manual/bash.html) · [Shell scripting for beginners — freeCodeCamp](https://www.freecodecamp.org/news/shell-scripting-crash-course-how-to-write-bash-scripts-in-linux/)

### music-mcp (by pedrocid)
The open-source MCP connector at the heart of this project. It bridges Claude Desktop and the macOS Music app using AppleScript. When Claude calls a tool (search for tracks, create a playlist, add songs), the connector translates that call into the appropriate AppleScript commands and sends them to the Music app. Results — track names, confirmation, errors — are returned to Claude, which then replies in chat.

Source and documentation: [github.com/pedrocid/music-mcp](https://github.com/pedrocid/music-mcp) · [npm registry: @pedrocid/music-mcp](https://www.npmjs.com/package/@pedrocid/music-mcp)

### MCP (Model Context Protocol)
The open standard that defines how Claude Desktop and connectors communicate. It specifies the format of tool calls, responses, and errors so that any MCP-compatible connector works with any MCP-compatible host. Connectors are started as separate processes and communicate with the host over standard input/output.

Learn more: [modelcontextprotocol.io](https://modelcontextprotocol.io) · [Anthropic MCP announcement](https://www.anthropic.com/news/model-context-protocol)

### Node.js + npx
The runtime required to execute the music connector. npx downloads and runs the connector from the npm registry on demand, meaning the connector is always up to date and no separate installation or update step is needed. The installer checks for Node.js 18+ and installs Node.js 22 LTS if it is absent or outdated.

Learn more: [nodejs.org](https://nodejs.org) · [What is Node.js? — freeCodeCamp](https://www.freecodecamp.org/news/what-is-node-js/) · [npm docs: npx](https://docs.npmjs.com/cli/v10/commands/npx)

### Python 3 (config merging)
Python ships with macOS and is used by the installer to read, modify, and write back Claude Desktop's JSON configuration file. It safely handles edge cases — malformed JSON, missing keys, first-time setup — and creates an automatic backup if it finds a file it cannot parse. Python was chosen over simpler text tools because JSON structure must remain valid, and text manipulation with `sed` or `awk` is fragile when applied to structured files.

Learn more: [python.org](https://python.org) · [Python for beginners](https://www.python.org/about/gettingstarted/)

### AppleScript
The macOS-native scripting language that the music connector uses to communicate with the Music app. AppleScript can tell the Music app to search for a track, create a playlist, add a song, and return results. It is the only reliable way to programmatically control the Music app on macOS — there is no public API. This is also why the macOS Automation permission is required: AppleScript requires explicit user consent before one app can control another.

Learn more: [AppleScript overview — Apple Developer](https://developer.apple.com/library/archive/documentation/AppleScript/Conceptual/AppleScriptLangGuide/introduction/ASLR_intro.html) · [AppleScript for absolute beginners](https://macosxautomation.com/applescript/firsttutorial/index.html)

### macOS disk image (.dmg)
The packaging format used to deliver the installer. A `.dmg` mounts as a read-only volume containing the installer file and this README. It is a standard macOS distribution format — familiar to users, code-signable, and self-contained. After setup, the `.dmg` has no further purpose and can be ejected and deleted.

## What the installer does — step by step

The installer runs five steps in sequence. Each one is described below in plain terms, along with why it exists.

---

### Step 1 of 5 — Checking your system

**What happens:** The installer checks two things: that you are running macOS (not Linux or Windows), and that you are on an Apple Silicon Mac (M1 or later, not Intel).

**Why:** The music connector communicates with the Music app using AppleScript, which is a macOS-only technology. There is no equivalent on other operating systems. The Apple Silicon restriction exists because the installer only bundles a Node.js package for the `arm64` architecture; Intel support was out of scope for v1. The installed macOS version is also read and printed to the terminal as a confirmation.

**What you see:**
```
▶ Step 1 of 5 — Checking your system
  ✓ Running macOS 15.3.2 on Apple Silicon
```

If you are on an Intel Mac, the installer stops here with a clear error message.

---

### Step 2 of 5 — Checking for Claude Desktop

**What happens:** The installer checks that `Claude.app` exists in your `/Applications` folder. If Claude Desktop is currently running, a warning is printed so you know a restart will be needed at the end.

**Why:** The music connector is configured inside Claude Desktop's settings file. There is nothing to configure if the app is not installed. The running-process check (`pgrep -x "Claude"`) is advisory only — it does not stop the installation, but it ensures you are not surprised when you need to restart at the end.

**What you see:**
```
▶ Step 2 of 5 — Checking for Claude Desktop
  ✓ Claude Desktop is installed
  ⚠  Claude Desktop is currently open. It will need to be restarted
     after this installer finishes for the changes to take effect.
```

If Claude Desktop is not installed, the installer stops and provides the download URL (claude.ai/download).

---

### Step 3 of 5 — Checking for Node.js

**What happens:** The installer checks whether Node.js is installed and whether the installed version is 18 or later. If Node.js is absent or outdated, it downloads the Node.js 22 LTS installer from nodejs.org, installs it (this step requires your Mac password), and confirms the result.

**Why:** The music connector is a Node.js package. Node.js must be present for `npx` to work. The installer checks for Node.js 18+ rather than a specific version so that users who already have a recent Node.js installation are not asked to install again.

**What you see (if Node.js is already installed):**
```
▶ Step 3 of 5 — Checking for Node.js
  ✓ Node.js v22.12.0 is already installed
```

**What you see (if Node.js needs to be installed):**
```
▶ Step 3 of 5 — Checking for Node.js
  Node.js is not installed. Installing now...
  Downloading the Node.js installer from nodejs.org...
  Installing Node.js — your Mac password is required...
  ✓ Node.js v22.12.0 installed successfully
```

The downloaded installer package is cleaned up from `/tmp` automatically after installation.

---

### Step 4 of 5 — Connecting the music server to Claude Desktop

**What happens:** The installer writes a music server entry into Claude Desktop's configuration file at `~/Library/Application Support/Claude/claude_desktop_config.json`. It handles three scenarios:
- **No config file exists:** Creates a minimal valid JSON file from scratch with the music entry.
- **Config file exists and is valid:** Uses Python to parse the file, add the music entry under `mcpServers`, and write it back — preserving any other servers already configured.
- **Config file exists but is malformed:** Creates a timestamped backup of the broken file, then creates a fresh config. The backup path is printed to the terminal.

After writing, a Python one-liner verifies that the resulting file is valid JSON.

**Why:** The installer must not destroy any other connectors the user has already set up. Python is used instead of text manipulation precisely because JSON has strict formatting rules — a single stray comma or missing brace makes the entire file unreadable by Claude Desktop. The JSON validation step at the end catches any unexpected write errors before the user restarts Claude.

**The entry written:**
```json
{
  "mcpServers": {
    "music": {
      "command": "npx",
      "args": ["@pedrocid/music-mcp@latest"]
    }
  }
}
```

**What you see:**
```
▶ Step 4 of 5 — Connecting the music server to Claude Desktop
  Existing Claude configuration found. Adding music server...
  ✓ Music server added to your existing configuration
  ✓ Configuration file verified — JSON is valid
```

---

### Step 5 of 5 — Opening the macOS permission screen

**What happens:** The installer prompts you to press Enter, then opens System Settings directly to Privacy & Security → Automation using a deep-link URL (`x-apple.systempreferences:com.apple.preference.security?Privacy_Automation`).

**Why:** macOS requires users to personally grant any application the right to control another application via AppleScript. This privacy protection is intentional and cannot be bypassed by a script. The installer opens exactly the right screen in System Settings so you do not have to navigate there manually.

**What to do in System Settings:**
1. Find **Claude** in the Automation list
2. Switch on the toggle next to **Music**

If Claude does not appear in the list yet, it will appear after the first time you use a music tool in Claude Desktop. Return to System Settings → Privacy & Security → Automation at that point and enable the toggle.

**What you see:**
```
▶ Step 5 of 5 — Opening the macOS permission screen
  One permission must be granted manually.
  ...
  ✓ System Settings opened to Privacy & Security → Automation
```

---

### After the installer completes

Two things remain for you to do:

1. **Grant the permission** in System Settings (enable the Music toggle under Claude in the Automation pane).
2. **Restart Claude Desktop** — quit it and reopen it from your Applications folder. The music tools become active once Claude restarts and loads the updated configuration.

## Using Claude with Apple Music

Once setup is complete, open Claude Desktop and describe what you want in plain English. No special syntax is needed. Some starting points:

**Sampling and hip hop**
- "Build a playlist of the 20 most sampled songs from the 1970s that were sampled in 80s and 90s hip hop. Call it 70s Hip Hop Samples."
- "Create a playlist of all the funk records that Dr. Dre sampled on The Chronic."
- "Make a playlist of James Brown tracks that were sampled in N.W.A songs."

**Folk and traditional lineage**
- "Build a playlist tracing the folk ballads that Bob Dylan drew from in his early career, starting with their traditional origins."
- "Create a listening guide to the Appalachian and Celtic sources behind American bluegrass — one traditional version and one modern interpretation of each tune."

**Classical influence**
- "Make a playlist of Beethoven's lesser-known works that directly influenced Brahms, paired with the Brahms pieces they shaped."
- "Build a playlist of the Hungarian and Romanian folk songs Bartók collected in the field, alongside his compositions that drew from them."

**Getting Apple Music links alongside the playlist**

Add this phrase to any prompt to receive clickable Apple Music links for every track — whether or not it is in your library:

*"For each selection add a URL to link to the track in Apple Music."*

This lets you preview tracks you do not yet own, add them to your library from the Apple Music catalogue, and then ask Claude to include them in the playlist.

## The library limitation — explained

This is the most important constraint to understand before using the music tools.

The music connector can only work with songs that are **already in your personal Apple Music library**. When Claude asks it to find a track, it searches your local library using AppleScript. It cannot browse or search the Apple Music streaming catalogue — Apple does not provide a public API for this, and AppleScript has no access to the catalogue.

**In practice this means:**

When you ask Claude to build a playlist, it will:
1. Build the list of tracks based on its knowledge of music history — the "correct" answer to your request
2. Search your library for each track
3. Add the ones it finds and create the playlist
4. Tell you which tracks it could not find

For users with large Apple Music libraries, most requests will be fulfilled well. For users with smaller libraries, Claude's response will include a list of missing tracks alongside their Apple Music links so you can add them before asking Claude to include them.

**This is by design, not a bug.** The playlist Claude suggests is always musically and historically accurate — the limitation is only which of those tracks it can add automatically.

## Managing the connector

**To disable the connector temporarily** — inside Claude Desktop, click the `+` icon in the chat input bar, go to Connectors, and toggle the music connector off. No restart required. Toggle it back on to re-enable.

**To disable by editing the config file** — quit Claude Desktop, open `~/Library/Application Support/Claude/claude_desktop_config.json` in a text editor, remove the `"music"` block under `"mcpServers"`, save, and reopen Claude Desktop.

**To re-enable after a manual edit** — run the installer again. It adds the music entry back without affecting anything else in the file.

**To remove the Automation permission** — go to System Settings → Privacy & Security → Automation, find Claude, and switch off the Music toggle.

## Troubleshooting

**"Claude Desktop is not installed"**
Download Claude Desktop from claude.ai/download, install it, sign in, and run the installer again.

**Node.js installation fails**
Download Node.js manually from nodejs.org (choose the LTS version for macOS Apple Silicon), install it, and then run the installer again.

**Claude is not listed in System Settings → Automation**
The Automation entry only appears after Claude has attempted to use the music tools at least once. Open Claude Desktop, type "Use the music info tool," then return to System Settings → Privacy & Security → Automation and enable the Music toggle under Claude.

**Music tools do not appear in Claude after setup**
Make sure you have fully quit and reopened Claude Desktop. If the tools still do not appear, open `~/Library/Application Support/Claude/claude_desktop_config.json` and confirm it contains a `"music"` entry under `"mcpServers"`. If the entry is missing, run the installer again.

**The installer stopped with a "malformed JSON" backup message**
The installer found an existing Claude Desktop config file that was not valid JSON — likely from a previous manual edit. It has saved a backup with a timestamp in the same folder (e.g. `claude_desktop_config.json.backup-20260405143022`) and created a fresh config containing only the music entry. Any other connectors you had previously configured will need to be added back. You can refer to the backup file to recover their entries.
